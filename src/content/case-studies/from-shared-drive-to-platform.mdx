<ProblemBlock>

Instructional designers are creative problem-solvers. We build Quality Assurance (QA) checklists when Storyline's spellcheck falls short. We design translation trackers when workflows break. We prototype workarounds when tools don't quite do what we need. The problem isn't a lack of solutions. It's that our best ones live in silos. Tucked away in spreadsheets, shared drives, and personal folders. Hard to share. Harder to maintain.

Our team relied on passing scrappy tools around to speed up tedious work. Each one was a one-off. Sharing meant downloading code from someone's shared drive and running it locally. Updates were verbal: *"hey, grab the new version."* Half the team always ended up on a stale copy. We needed a system that made the technical power invisible to the end user.

</ProblemBlock>

<ProcessBlock>

I built the Learning Experience Design (LXD) Hub. A centralized, authenticated portal for learning and development teams. A single dashboard where designers access automation tools in seconds, each appearing as a clean clickable card. Sign in once, every app launches from the browser. No installation. No version confusion. No technical hoops.

From day one, the focus wasn't the code. It was the user experience. Instructional designers shouldn't need to learn command-line syntax to use a useful tool. They need something that behaves like good learning experiences: intuitive, accessible, friction-free.

Each tool is its own self-contained web app, so we can update or replace any one of them in isolation without impacting the rest.

<OutcomeBlock metrics={[

Before the LXD Hub, our team spent nearly as much time setting up tools as using them. Translation was the cleanest example: requests previously went to one team member, who ran the local version of a translation script. The standard service-level agreement was three days per file.

With the Hub, requestors manage their own translation work from submission to completion, in two to four hours. That's not just faster delivery, it's an entire administrative tier removed from the team's plate. Nineteen source files moved through in a single recent week with no one being the bottleneck.

</OutcomeBlock>

<LessonsBlock>

Technology doesn't necessarily empower people. Design does. By focusing on usability and centralization, we turned scattered, hard-to-maintain scripts into a cohesive ecosystem that supports creative work instead of distracting from it. When automation becomes effortless, instructional designers can finally spend their energy where it matters: building learning experiences worth shipping.

</LessonsBlock>
