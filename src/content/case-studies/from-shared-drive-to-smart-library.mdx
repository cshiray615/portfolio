<ProblemBlock>

Have you ever opened a team shared drive and instantly lost faith in humanity?

My team managed training content in six languages, most of it authored in Articulate Storyline. We had guidelines for naming conventions and file locations. We had standards. But standards drift year over year, and "drift" is the polite word for it. Finding the latest version of a file was, charitably, archaeology.

So I set out to build a better system.

</ProblemBlock>

<ProcessBlock>

I approached this the way I approach learning design: start with the user, clarify the workflow, prototype. I mapped how instructional designers actually search, upload, and update content, paying attention to the needs no generic Content Management System (CMS) accounts for. Then I translated those pain points into system behaviors: version control, metadata tagging, linking to Review 360 and the Learning Management System (LMS), and auto-generated course descriptions.

The result is The Vault. A smart CMS built specifically for training files — PowerPoint, Storyline, and the metadata around them. Designed for how learning teams think about content, not how a generic file server does.

Chaotic training-content storage became a structured, searchable, auditable library. Every version of every file is automatically saved (no more guessing whether `final_final_v2` was the right one). Check-in/check-out prevents the silent overwrites that used to eat our lunch.

The feature the team uses every single day, and tells me they couldn't go back without, is the **check-out impact preview**: the moment you check a file out to edit, The Vault shows you every course and module that file is attached to, so you can see exactly what your change will affect *before* you save it. That single behavior turned a category of small disasters into a non-event.

<Appendix title="What's coming next">

Now that The Vault securely stores every version of our Storyline and PowerPoint source files, the next step is to make those files *smarter*. The Vault and the LXD Hub already share the same AWS backbone, so the plan is to connect the two into one ecosystem.

The goal: every time a new version of a file is uploaded, it triggers automated quality checks powered by AI.

- **Automated QA Scoring** — each upload gets scanned for missing alt text, inconsistent fonts, broken links, and accessibility gaps, using the same Quality Assurance (QA) logic built into our Storyline inspection tools.
- **Content Search Intelligence** — text extracted from source files feeds a searchable content index. No more *"which file mentions blue tape?"* moments.
- **AI-Powered Summaries and Tags** — as files are processed, The Vault auto-generates descriptions and metadata, reducing manual data entry and improving searchability.

Together these upgrades turn The Vault from storage into an intelligent CMS that actively supports design work: spotting issues, surfacing insights, reducing maintenance.

</Appendix>

<LessonsBlock>

Building these applications reinforced a simple truth: good software design and good learning design share the same DNA. Both start by understanding human behavior. How people search, click, and get stuck. Then they design systems that remove the friction so users can concentrate on what actually matters. For instructional designers, that's designing learning experiences. For me, that's building the system that gets out of their way.

</LessonsBlock>
